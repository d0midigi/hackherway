Kerberoasting

Offline password extraction attack abusing Kerberos service tickets

Overview

Kerberoasting is an attack technique that abuses Kerberos service principal names (SPNs) to request service tickets that can be cracked offline.

Attack Preconditions

  • Valid domain user account
  • Service accounts with weak passwords
  • SPNs registered in Active Directory

Attack Flow

  1. Enumerate service accounts with SPNs
  2. Request TGS tickets from the KDC
  3. Extract ticket hashes
  4. Perform offline cracking

Detection Opportunities

  • High volume of TGS requests
  • Service ticket requests from atypical hosts
  • Event ID 4769 anomalies

Mitigations

  • Use Managed Service Accounts
  • Enforce long, random passwords
  • Monitor Kerberos service ticket activity
