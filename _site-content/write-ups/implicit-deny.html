What is Implicit Deny?

Access Control Lists (ACLs) are fundamental security features in Cisco routers that provide powerful packet filtering capabilities. These ordered lists of rules examine incoming and outgoing network traffic based on specific criteria, allowing administrators to control and filter traffic according yo predefined policies.

║Implicit Deny

A critical concept in ACLs is the Implicit Deny rule. This rule operates as an unspoken final command that denies all traffic not explicitly allowed by the preceding rules. In Cisco ACL configurations, while this rule is not visible in the configuration, it is always active. The presence of Implicit Deny ensures that traffic is allowed only if it matches specified rules, effectively implementing a whitelist approach.

For example:

permit ip 192.168.1.0 0.0.0.255 any
permit tcp any 10.0.0.0 0.0.0.255 eq 80

Any traffic that does not match the above rules will be denied because of the implicit deny at the end of the list.

║Deny Any Any Rule

The Deny any any command is a variation of the implicit deny. This rule explicitly specifies that any packet not matching a prior rule is denied. It is often included at the end of ACLs for clarity and logging purposes.

In Extended ACLs, where traffic filtering involves both source and destination IPs and other attributes, the Deny any any rule includes the keyword "any" twice, covering all traffic from any source to any destination. For example:

deny ip any any

This command ensures no unintended traffic slips through.

║Key Differences Between Implicit Deny and Deny Any Any:

  • Implicit Deny: Always present by default but not displayed in the configuration.

  • Deny Any Any: Explicitly configured by the administrator, often used for logging and troubleshooting to document dropped traffic.

By leveraging these mechanisms, administrators can tightly control network access, safeguarding against unauthorized use while maintaining operational clarity.
